# MonsterSurvival

## 1. 게임 컨셉

### High Concept
* 끊임없이 몰려오는 적들을 쓰러트리면서 살아남는 게임

### 핵심 메카닉
* 모든 공격은 자동으로 이루어지며 플레이어는 이동 조작에만 집중한다.
* 수많은 적들이 플레이어를 향해 군집을 이루며 다가온다.
* 플레이어는 공격으로 적을 쓰러트리면 그 즉시 경험치를 획득하고 레벨이 오를때마다 무기가 추가되거나 강화된다.

---

## 2. 개발 범위

### 플레이어 1종
* 사용자의 입력에 따라 그 방향으로 이동하며 레벨 업 할때마다 무기가 강화되거나 추가된다.

### 무기 3종
* 자동 타켓팅 방식의 무기 1종
* 일정 시간마다 플레이어 주위를 강하게 베는 무기 1종
* 일정 시간마다 플레이어 주위를 n초 동안 빙빙 돌며 공격하는 무기 1종

### 몬스터 3종
* 플레이어를 향해 다가오는 일반몬스터
* 이동속도가 빠르지만 체력이 낮은 몬스터
* 이동속도가 느리지만 체력이 높은 몬스터

### UI 요소
* 타이틀 화면 (게임시작, 게임 종료)
* 인게임 화면
* 팝업 화면 (무기 선택 or 무기 강화)
* 게임 오버 화면 (Restart 버튼, Back Title 버튼)

## 3. 예상 게임 실행 흐름

### 타이틀 화면
* 첫 게임을 실행하면 타이틀 화면이 나타난다. 타이틀 화면에는 게임 로고와 Game Start 버튼, Game Exit 버튼 2개가 존재한다.

### 인게임 화면
* Game Start 버튼을 누르면 인게임 화면으로 전환된다.

### 팝업 화면
* 인게임 화면에서 적을 쓰러트려 레벨이 오르면 팝업 화면이 나타난다. 팝업화면은 무기를 선택하거나, 강화하면 종료되고 인게임 화면으로 돌아간다.

* 플레이어의 체력이 0이 되면 게임 오버 화면으로 전환되고 게임을 재시작하거나, 타이틀 화면으로 돌아간다.

## 4. 개발 일정

### 1주차
* 프로젝트 생성 및 타이틀 화면 버튼 및 view 배치 -> 화면 전환 개발

### 2주차
* 플레이어 객체 생성 및 이동 구현

### 3주차
* 몬스터 설계 및 추격 AI구현, 오브젝트 풀링 방식을 이용한 몬스터 다량 생성 관리

### 4주차
* 무기 3종 작동 구현

### 5주차
* 몬스터에 의한 피격 및 플레이어의 공격에 의한 몬스터의 피격과 넉백 구현

### 6주차
* 경험치와 레벨 구현 및 팝업 화면 구현

### 7주차
* 게임 오버 화면 구현 및 화면들간의 연동 구현

### 8주차
* 버그 수정
